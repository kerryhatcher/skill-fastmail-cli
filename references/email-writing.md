# Email Writing

Use this reference when drafting, replying to, or forwarding email so the copy sounds natural.

## Goal

Write like a real person handling real email.

- Clear
- Direct
- Context-aware
- Specific when details are available
- Natural enough that it sounds like something the user would actually send

Do not write like a marketing page, customer support template, or generic AI assistant.

## Default Style

- Prefer short, plain words
- Use contractions: `I'm`, `I'll`, `don't`, `can't`, `it's`
- Keep sentences varied in length
- Sound like an actual person, not a polished narrator
- Match the user's likely tone: brief for routine email, warmer for personal notes, firmer when the situation calls for it

## Avoid Common AI Tells

Avoid or heavily limit words and phrases like:

- `delve`
- `landscape`
- `multifaceted`
- `nuanced`
- `robust`
- `leverage`
- `utilize`
- `seamless`
- `foster`
- `pivotal`
- `paramount`
- `holistic`
- `innovative`
- `streamline`
- `transformative`
- `It's important to note that`
- `In today's rapidly evolving landscape`
- `A testament to`
- `navigate the complexities`
- `cannot be overstated`
- `embark on a journey`
- `Moreover`
- `Furthermore`
- `Additionally`
- `Nevertheless`
- `Consequently`
- `Accordingly`

Also avoid these structural tells:

- overly even sentence lengths
- empty filler that sounds thoughtful but says little
- generic praise with no concrete point
- hedging every statement into bland neutrality
- em dash overuse
- list-heavy formatting when a normal paragraph would do

## What To Do Instead

- Say the thing plainly
- Use the facts from the thread
- Make the ask or answer obvious
- Leave out filler
- End once the message is done

Good replacements:

- `use` instead of `utilize`
- `help` instead of `facilitate`
- `start` instead of `commence`
- `show` instead of `demonstrate`
- `but`, `and`, `also`, `still`, `so` instead of formal transitions

## Tone By Context

### Routine work email

- brief
- useful
- low-drama

Example:

```text
Hi Sam,

I saw this come through. I don't have the final answer yet, but I'll be there and can help once the plan is locked in.

Just let me know what you need me to cover.

Thanks,
Kerry
```

### Friendly / collaborative reply

- warmer
- still concise
- can sound a little informal

Example:

```text
Hi Jen,

Thanks for sending this over. I took a look and it makes sense to me.

I can handle my part this afternoon. If anything changes, send me a note.

Best,
Kerry
```

### Firm or corrective email

- direct
- calm
- no extra polish

Example:

```text
Hi Chris,

I need to correct one part of this. The deadline wasn't moved, so we're still working toward Friday.

If that changes, I'll let you know right away.

Kerry
```

## Specificity Rules

Use concrete details whenever they exist in the thread:

- dates
- times
- names
- commitments
- open questions
- the exact thing being asked for

Bad:

```text
Thank you for your email. I appreciate the update and look forward to collaborating further.
```

Better:

```text
Thanks for the update. I don't have the final service details yet, but I'll be there for all of them and can help wherever needed.
```

## Final Pass Before Creating A Draft

Check the message for these problems:

- Does it sound like something the user would actually say?
- Can any sentence be shorter?
- Is there filler that can be cut?
- Are there formal transition words that can be replaced or removed?
- Does it use concrete thread details where useful?
- Does it avoid obvious AI-favorite words and phrasings?

If a sentence sounds like a press release, rewrite it.
